# Web Interface Guidelines Review Skill

## Purpose

Use this skill after a landing or UI implementation exists. It is a review gate, not a planning skill.

## Review checklist

- Does the page use semantic HTML landmarks and headings?
- Are CTAs keyboard-accessible and visually distinct?
- Does the layout work at common desktop and mobile widths?
- Is contrast acceptable for dense trading content?
- Are motion effects subtle and disable-safe?
- Are loading, error, disabled, and pending states represented where relevant?
- Is the page fast enough for a landing/demo context?
- Are claims truthful and consistent with legal/product docs?
- Is there no secret, credential, or live trading logic in the landing app?

## Required output

When reviewing, produce:

1. Critical issues blocking merge.
2. Important improvements.
3. Optional polish.
4. A final merge/no-merge recommendation.
