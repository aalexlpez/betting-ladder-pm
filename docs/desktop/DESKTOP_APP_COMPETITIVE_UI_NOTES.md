# Desktop App Competitive UI Notes

## Purpose

Keep desktop design grounded in the competitive landscape without copying competitor UI.

## What competitors validate

### Traderline

Validates:

- ladder-first terminal positioning;
- desktop app distribution;
- one-click execution as a core promise;
- users may compare us directly against Betfair-style workflows.

Implication:

- our UI cannot be a generic dashboard; it must look immediately execution-focused.

### Strata

Validates:

- local-first desktop narrative;
- API credentials and trading data staying local as a trust signal;
- advanced traders value desktop terminal posture.

Implication:

- our UI should show credential/gate status without exposing secrets.

### Polyblock / PolyTraderPro / native UIs

Validate:

- demand for faster Polymarket tools;
- multi-market views, terminal UX, and speed claims are already part of the market.

Implication:

- we should differentiate by risk-gated ladder execution, clarity, and no-custody posture rather than broad automation claims.

## UI differentiation wedge

Prediction Ladder should be:

- narrower than multi-feature terminals;
- stronger at ladder execution clarity;
- safer-looking around live mode;
- more transparent about gates and audit logs;
- credible as a controlled live-ready tool.

## Avoid

- claiming to be first or only;
- copying competitor layouts;
- using competitor logos/screenshots without permission;
- attacking competitors;
- pretending paper/demo behavior is live execution.

## Preferred comparison language inside app

The desktop app itself should not include aggressive competitor comparisons. Competitive positioning belongs mostly on the landing page and docs. Inside the app, show product strengths through UI:

- visible gates;
- visible audit trail;
- ladder clarity;
- explicit execution modes;
- local-first credential posture.
