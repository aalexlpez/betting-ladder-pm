# Landing Page Strategy

## Goal

Create a landing page that makes the Prediction Ladder product look credible, specific, and commercially differentiated from current prediction-market terminals.

This is not a generic SaaS landing page. It is the public face of a live-ready Windows desktop trading terminal.

## Primary audience

Advanced or semi-advanced prediction-market traders who understand order books and want a faster manual execution workflow than native Polymarket/Kalshi interfaces.

## Primary message

> A Windows desktop ladder terminal for prediction markets: fast manual execution, live market data, local-first credential posture, and risk-gated live trading.

## Strategic wedge

The market already has emerging competitors. The page should not claim we are the first. It should claim a sharper scope:

- Desktop-first terminal, not a generic web dashboard.
- Ladder-first execution, not chart-first analytics.
- Manual trading workflow, not an opaque bot or AI signal product.
- No-custody posture and local credential handling.
- Explicit safety and legal gates before live trading.
- Paper/live-dry-run modes for QA, training, and fallback.

## CTA

Primary CTA:

> Request Windows beta access

Secondary CTA:

> View live-readiness checklist

If the app package exists, the primary CTA can become:

> Download Windows demo

## Tone

- Precise
- Professional
- Calm
- Safety-aware
- Trader-oriented

Avoid:

- Casino language
- Meme crypto energy
- Generic AI SaaS copy
- Profit guarantees
- Claims of legal certainty
- Fake testimonials or fake customer logos

## Conversion success

The landing page succeeds if a serious trader or evaluator can answer within one minute:

1. What is this product?
2. Who is it for?
3. Why is it different from existing tools?
4. How does it handle keys, funds, and live-trading risk?
5. What can I do next?
