# Landing Resources

## External inspiration categories

Use these categories for design research, not as copy sources:

- Trading terminals and execution platforms.
- Developer tools with technical trust messaging.
- Fintech/security landing pages that emphasize custody, credentials, compliance, and auditability.
- Desktop-app landing pages with clear download/waitlist flows.

## Useful design principles

- Show product before explaining all features.
- Use real product concepts instead of decorative dashboards.
- Explain security posture in plain language.
- Avoid fake social proof.
- Make the primary CTA match the current release state.
- Keep legal language accurate and visible.

## Skill policy

The external `frontend-design` skill was evaluated as useful because it focuses on distinctive, production-grade UI rather than generic AI-generated aesthetics. For this repository, its principles are represented by the local `.agents/skills/frontend-design` skill and combined with a product-specific `landing-conversion-strategist` skill.
