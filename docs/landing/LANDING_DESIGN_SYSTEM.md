# Landing Design System

## Visual direction

Professional trading terminal, not generic crypto SaaS.

Keywords:

- Dense but readable
- Dark terminal surface
- Calm confidence
- Precise typography
- Visible risk state
- Product-led hero

## Above-the-fold requirements

The first screen should contain:

- A short eyebrow.
- A direct headline.
- A concise subheadline.
- Primary and secondary CTA.
- A credible product mockup showing a ladder, not a fake generic chart.
- A visible live-mode/risk-gate cue.

## Product visual

The mockup should include:

- Price ladder rows.
- Bid and ask liquidity columns.
- Selected stake preset.
- Execution mode indicator: `paper`, `dry-run`, or `live gated`.
- Risk banner or live-gate status.
- Small order blotter or audit trail.

## Layout

Recommended layout:

- Hero: two-column desktop layout.
- Product visual: right side or full-width below hero.
- Value cards: 3-column grid.
- Wedge/comparison: compact table or cards.
- Safety section: high-contrast panel.
- Workflow: numbered steps.
- Final CTA.

## Color guidance

Do not hardcode a dependency on neon crypto visuals. Use a restrained palette suitable for a trading terminal. Colors should communicate state:

- Neutral/dark base for terminal surfaces.
- Positive/negative states for bid/ask only where useful.
- Warning state for live/risk gates.
- Accent color for CTA.

## Typography

Use clear sans-serif for marketing copy and a monospaced or tabular-number treatment for ladder data.

## Motion

Motion should be minimal. If used, it should suggest live market updates or terminal focus, not decorative noise.
