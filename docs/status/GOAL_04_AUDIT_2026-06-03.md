SDK/source-of-truth client runtime, WebSocket-first streaming, and unified

## Findings fixed during audit

- Kalshi market normalization no longer treats a missing `status` field as
  `open`; unresolved status now remains unknown and blocks resolution.
- Configured real-market fallback no longer reports `freshness: "fresh"` or
  `connectionMode: "snapshot"` because it is only a resolve request, not provider
  success.
- Market-search success wrappers now report `connectionMode:
  "polling_fallback"` consistently with discovery snapshots.
- `docs/ai/CODEX_GOALS.md` no longer asks Goal 04 to implement WebSocket runtime;
  that work is deferred to Goal 04B.
- `docs/architecture/SYSTEM_ARCHITECTURE.md` now reflects
  `TradableMarketRef`/`NormalizedOrderBookSnapshot` market-data ports instead of
  stale `MarketRef`/raw snapshot wording.
- `packages/market-data/README.md` now explicitly says official SDK/WebSocket
  runtime wiring is deferred to Goal 04B.

## Verified strengths

- Provider-specific Polymarket/Kalshi payload handling stays inside
  `packages/market-data/src/providerAdapters.ts` or `providerMetadata`.
- `packages/core` remains provider-neutral and has no network, SDK, env, React,
  or Tauri imports.
- `packages/market-data` depends only on `@prediction-ladder/core`.
- Desktop consumes normalized snapshot state but exposes no provider SDK,
  credential, shell, filesystem, or live execution path to the renderer.
- Fixture and configured fallback sources remain distinguishable from
  `official_live`.
- Tests cover Polymarket/Kalshi normalization into shared ladder-facing rows,
  malformed payloads, stale Polymarket snapshots, empty liquidity, unsupported
  markets, missing outcomes, invalid tick sizes, network errors, provider
  errors, credential-required responses, non-live source blocking, and disabled
  execution.

## Remaining debt / non-blockers at Goal 04 audit time

- Goal 04B later added official runtime documentation, WebSocket message
  normalization, stream/reconnect/freshness state helpers, renderer-safe
  projection, and the Tauri command names.
- Goal 04B did not make the React UI display a live ladder yet; that remains
  Goal 04C.
- The Rust/Tauri command handlers are still a secret-free boundary surface and
  do not yet own a live provider WebSocket transport.
- The desktop market surface still exposes provider surfaces as separate UI
  sections. Unified multi-venue market search/list is Goal 04C.
- No authenticated account/portfolio metrics exist in Goal 04; metrics remain
  explicitly unknown.
- Direct live endpoint smoke was partial in this environment: Polymarket market
  discovery returned HTTP 200; Kalshi market discovery could not complete from
  the local environment because the request returned `000` after TLS/network
  failure. This is not counted as adapter failure because deterministic
  official-shape fixture tests pass and official docs still support the endpoint.

## Official-source checks

- Polymarket docs confirm `GET https://gamma-api.polymarket.com/markets/{id}`.
- Polymarket docs confirm `GET https://clob.polymarket.com/book` with required
  `token_id` and timestamp/tick/order-book fields.
- Kalshi docs confirm `GET /markets` with `status` and `tickers` filters.
- Kalshi docs confirm `GET /markets/{ticker}/orderbook` returns
  `orderbook_fp.yes_dollars` and `orderbook_fp.no_dollars`; asks are implied by
  binary YES/NO reciprocity.
- Kalshi docs show authentication headers in the API reference, while separate
  market-data quick-start docs describe public REST reads. Goal 04 correctly
  maps real `401`/`403` responses to `provider_credentials_required`.
