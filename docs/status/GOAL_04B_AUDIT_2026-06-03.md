# Goal 04B Audit - 2026-06-03

Traceability/status artifact. Current phase and next action remain controlled by
`docs/ai/context-handoff.md`.

## Scope

Audited the Goal 04B official provider runtime and WebSocket-first streaming
work after implementation.

Routes used: architecture/package boundaries, provider adapters, Tauri command
boundary, legal/live safety gate, market-data and market-selection specs.

Official provider documentation was rechecked during the audit:

- Polymarket documents official TypeScript, Python, and Rust CLOB clients and a
  public market WebSocket channel at
  `wss://ws-subscriptions-clob.polymarket.com/ws/market`.
- Kalshi documents official Python and TypeScript SDKs, recommends
  REST OpenAPI and WebSocket AsyncAPI/direct integration as production
  source-of-truth, and requires authenticated WebSocket handshake headers for
  public market-data channels.

## Result

Verdict: **CONDITIONAL_PASS** for Goal 04B after audit fixes.

Goal 04B now has deterministic WebSocket message normalization, explicit stream
states, REST recovery helpers, renderer-safe projection, and registered Tauri
market-data command names. It is not yet the visible desktop market workflow;
that remains Goal 04C.

## Findings fixed during audit

- Provider-specific stream/runtime errors now preserve `providerId` in
  `providerMetadata` before being projected to the renderer. Kalshi malformed,
  auth-required, or runtime errors no longer default visually to Polymarket.
- The desktop terminal state now keeps provider-specific error states on the
  correct provider card instead of selecting Polymarket for every failed
  `MarketDataResult`.
- Active market-data documentation no longer says Goal 04B is the next
  implementation block after Goal 04B already landed.
- The older Goal 04 audit report now explicitly marks its "no runtime yet" debt
  as Goal-04-time context and points to this follow-up audit.

## Verified strengths

- `packages/core` remains pure and provider-neutral.
- `packages/market-data` still depends only on `@prediction-ladder/core`.
- WebSocket normalizers keep raw provider payloads behind adapter/runtime code
  and return normalized order-book snapshots.
- Kalshi WebSocket access remains a credential gate; missing credentials are not
  treated as connected or successful.
- Renderer projection strips provider metadata and does not expose SDK objects,
  WebSocket objects, auth headers, signing payloads, credentials, or secrets.
- Tauri command registration remains limited to `app_get_status`,
  `market_search`, `market_get_order_book`, and `market_subscribe`.
- No live order placement, cancellation, account balances, positions, geoblock
  workaround, bots, strategies, copy trading, or AI signals were added.

## Remaining debt / non-blockers

- The Rust/Tauri command handlers are still secret-free boundary stubs. They do
  not yet instantiate the TypeScript `OfficialMarketDataRuntime` or a Rust-side
  provider WebSocket transport.
- The React renderer is not wired to call `market_search`,
  `market_get_order_book`, or `market_subscribe`; opening
  `http://localhost:1420/` still shows terminal state, not live ladder data.
- Polymarket streaming is implemented as official documented WSS message
  normalization and transport injection, not as a concrete live socket running
  in the desktop process.
- Kalshi streaming normalization exists, but a real Kalshi connection remains
  blocked until a Tauri-side/local credential provider can create official
  handshake headers.
- Search-all-provider runtime behavior still aborts on the first provider error;
  Goal 04C should decide whether the UI needs partial provider results with
  per-provider error rows.
- Authenticated account/portfolio metrics remain unavailable and must stay
  unknown in the UI.
